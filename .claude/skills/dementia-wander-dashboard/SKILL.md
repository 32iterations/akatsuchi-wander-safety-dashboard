# 失智症徘徊預警 3D Dashboard Skill

## 使用時機

當使用者要求：

- 強化 3D 數位分身（React Three Fiber）效果
- 設計或修改與徘徊相關的 Rive 動畫區塊
- 規劃館內平面圖 / 空間利用率的 UI 卡片
- 為評審或照護團隊撰寫 Demo 解說腳本

時，優先啟用此 Skill。

## 核心指引

1. **以評審與照護人員為優先讀者**
   - 所有視覺元素必須一眼可懂，避免過度炫技但難以解釋。
   - 任何 3D 互動都應詮釋為臨床或照護上的價值（例如：更快找到長輩）。

2. **3D 場景設計**
   - 盡量維持既有 `DigitalTwinCanvas` 的結構。
   - 若需新增物件，請保持：
     - 出入口明顯（例如用亮色或圖示標記）。
     - 護理站或照護人員位置有清楚的地標。
   - 避免讓場景變得太複雜，以免評審分心。

3. **Rive 動畫協調**
   - 對應風險等級（low / medium / high）使用不同節奏與顏色。
   - 在註解中清楚說明：前後端如何根據風險分數切換 Rive 狀態。

4. **空間利用率與事件回放**
   - 優先提供「一眼看出擁擠與否」的視覺（如簡單色條、標註人數）。
   - 事件回放畫面要能說清楚：
     - 從什麼時間點開始徘徊？
     - 何時觸發警示？
     - 介入後狀況如何改善？

## 範例流程

- 「請幫我把 3D 場景調整成三層樓版本」：
  1. 先讀 `components/scene/DigitalTwinCanvas.tsx`。
  2. 提出樓層分層策略（例如：使用 y 軸分層 + 顏色區分）。
  3. 再修改程式碼，並留下中文註解說明每一層代表的空間。

- 「請幫我為這個 demo 撰寫 2 分鐘投影片腳本」：
  1. 閱讀 `app/page.tsx`、`app/admin/page.tsx` 文字敘述。
  2. 以「問題 → 解法 → Demo 流程 → 擴充性」為架構產出腳本。
